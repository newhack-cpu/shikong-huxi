### 致命问题（必须修，否则丢国奖）❌

#### 问题 1：MSTN 输给 LightGBM 一个数量级，作为"核心创新"立不住

| 模型 | MAE | R² |
|---|---|---|
| LightGBM | **0.5614** | 0.9995 |
| MSTN（你的方法） | 12.20 | 0.9488 |
| 提升倍数 | **22 倍** | — |

**评委角度**：你的核心创新模型，被一个 2017 年的传统算法以 22 倍精度碾压，"创新"何在？  
**真正原因**：LightGBM 直接用 `pm25_lag_1h`（前一小时PM2.5）作为输入，本质等于"用 99 时刻预测 100 时刻"，是"伪精度"——不是模型好，是特征"作弊"。

**必须解决**（任选其一或组合）：
- **方案 A（推荐）**：去掉 lag_1h、lag_2h 等"近邻泄漏"特征，重做对比实验。MSTN 大概率反超或持平 LightGBM。
- **方案 B**：把对比改成"多步预测"（预测未来 6h / 24h），在长程预测上 MSTN 优势会显现。
- **方案 C**：保留两套结果但重新讲故事——"短期1h用LightGBM足够，长期/多城市用 MSTN"。

#### 问题 2：MSTN 的"空间卷积"名实不符

代码 `6_advanced_model.py` 第 158-171 行：
```python
class SpatialConvolution(nn.Module):
    def forward(self, x):
        x = F.relu(self.bn1(self.conv1(x)))   # x 形状 [batch, hidden, time]
```
**真相**：这个"空间"卷积实际是沿"特征维"的 1D 卷积，没有任何"空间"含义。单城市数据没有空间维度。

**评委一看代码就懂这是"空间建模"包装下的特征聚合**。这是命名上的不诚实，国一答辩问起来会被打。

**必须解决**：
- **方案 A**：把这个模块如实改名为 `FeatureAggregation` / `LocalContextConv`，承认它是"特征局部聚合"。
- **方案 B（更好）**：真的把"空间"做出来——跨城市/跨站点。需要拉真实多站点数据 + 修改输入张量为 [batch, time, station, feature]。

#### 问题 3：多城市数据是基于北京数据的"乘法+噪声"模拟

`7_multi_city_fusion.py`：
```python
df_tianjin['pm25'] = df_tianjin['pm25'] * 0.9     # 天津
df_shijiazhuang['pm25'] = df_shijiazhuang['pm25'] * 1.3  # 石家庄
```
**问题**：这违反"作品应来源合规"的基本要求，国赛评委一定会抽查代码。

**必须解决**：
- **方案 A**：把"多城市"改名为"北京多站点扩展"，并在报告/PPT如实说明。
- **方案 B（更好）**：去拉真实的 OpenAQ / 中国空气质量在线监测发布平台 数据。

#### 问题 4：消融实验是写死的硬编码数字

`8_ablation_study.py` 第 244-251 行：
```python
simulated_results = {
    'full':         {'mae': 12.20, 'rmse': 16.43, 'r2': 0.9488},
    'no_attention': {'mae': 14.31, ...},
    ...
}
results = simulated_results.get(model_variant, ...)  # 直接查表返回
```
**真相**：消融实验根本没在跑模型，是一个 hardcode 的字典查询。

**必须解决**：把每个消融变体真的训一次。这才是消融实验。

#### 问题 5：BHI 公式经不起推敲

```python
BHI = 0.65 × PM2.5/5 + 0.20 × 温度不适度 + 0.15 × 风险趋势
```

- PM2.5 系数 0.65、归一化除以 5（即 PM2.5=300 时该项=39，几乎吃满 65 权重的 60%）
- 等价于 `BHI ≈ 0.13 × PM2.5 + 微调`
- 温度阈值 18-22°C 是写死的"舒适区"，没有引用任何医学/环境健康标准
- "风险趋势"用过去3小时差分均值，逻辑上和当下浓度耦合度高

**评委可能问**："你这个 BHI 和 AQI 有什么本质区别？为什么权重是 0.65/0.20/0.15？"

**必须解决**：把权重改成有依据的（引用 WHO PM2.5 健康指南、GB 3095 环境空气质量标准），或者改成数据驱动的（用相关性分析得到权重）。

#### 问题 6：代码与报告/PPT 的数字不一致

| 位置 | 切分比例 |
|---|---|
| 报告 5.1 节 | 前 80% / 后 20% |
| `6_advanced_model.py` | 70% / 15% / 15% |
| `model_trainer.py` | 80% / 20% |

| 位置 | 数据规模 |
|---|---|
| PPT slide 3 | 43,824 条 |
| 报告 4.2 | 41,757 条（清洗后）+ 10万+ 条（多源） |
| 实际跑出来 | 取决于具体数据文件 |

**必须解决**：所有数字以 `paper_summary.txt` 为单一真实来源，统一报告 + PPT + 信息表 + 答辩稿。
